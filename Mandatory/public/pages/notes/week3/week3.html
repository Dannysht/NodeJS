Node modules

The node_modules directory is a folder that is automatically created when you install dependencies for a Node.js project using npm.
It contains all the packages and modules that your project depends on, including their dependencies, and is used by Node.js to resolve and load the required modules at runtime.
The node_modules directory is organized based on the package names and versions specified in the package.json file of your project.
Each package is installed in its own subdirectory within the node_modules directory, and can be accessed using the require function in your Node.js code.
To install dependencies like for example Express we use npm install express, which will create a node_modules directory with an express subdirectory containing
the express package and its dependencies. You can then load the express package in your code using require('express').
The node_modules directory should not be included in version control or deployed with your application, as it can be easily recreated by running npm install.
Instead, you should include the package.json file.

Static files in express

To use static files we use the following command express().use(express.static("Name of the folder")).
Doing this allows the user acess to a folder in which we store static files like HTML, JS and CSS files.
This is done to avoid security breaches only allowing the user to files they are allowed to view.
