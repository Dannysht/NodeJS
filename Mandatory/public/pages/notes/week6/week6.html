Client-side redirection vs. server-side redirection.

Client-side redirection, also known as client-side forwarding or JavaScript redirection, is a type of redirection that occurs within the user's web browser.
This is typically achieved by using JavaScript to modify the browser's current location or by using the HTML meta tag to redirect the browser to a new URL.
Client-side redirection is fast and responsive, as the redirection occurs immediately within the user's browser.
However, it is also less reliable, as it depends on the user's browser and JavaScript settings.

Server-side redirection, on the other hand, is a type of redirection that occurs on the server-side of a web application.
This is typically achieved by using server-side code, to redirect the user's browser to a new URL.
Server-side redirection is more reliable, as it does not depend on the user's browser or JavaScript settings.
However, it can also be slower and less responsive, as the user's browser must wait for the server to respond with the new URL.

Fetching in the client vs. fetching in the frontend

Fetching data in the client typically refers to the process of making HTTP requests to a server from within the client-side code of a web application,
such as JavaScript running in a user's web browser.
This allows the client-side code to dynamically load and update data from a server without requiring a full page refresh.

Fetching data in the frontend, on the other hand, refers to the process of retrieving data from a backend API
or server using server-side code.
The data is typically retrieved in response to a client-side request,
such as an AJAX request made using JavaScript.

Structuring HTML with semantic HTML

Structuring HTML with semantic HTML involves using HTML tags that convey meaning and describe the content of the page.
Semantic HTML helps search engines and other web crawlers understand the structure and content of the page,
making it easier to index and rank the page in search results. Examples of semantic HTML tags include header, nav, main, footer, etc.
These tags provide additional context and meaning to the content on the page, making it easier for both humans and machines to understand.
