Intro to Node.js

With Node.js, developers can use JavaScript on the server-side,
which was previously limited to client-side programming in web browsers.
Node.js provides a rich set of built-in modules and libraries that allow developers to handle various tasks
such as file system access, networking, and database interactions.
It also has a vast ecosystem of third-party modules and tools available through the npm package manager,
which can help streamline development and enhance the functionality of applications.

Variables and Data types

Data types

Number: represents both integer and floating-point numbers.

String: represents a sequence of characters.

Boolean: represents a logical entity and can have one of two values: true or false.

Null: represents a deliberate non-value or non-existent value.

Undefined: represents an uninitialized value or a variable that has been declared but not assigned a value.

Object: represents a collection of properties and can include other data types, functions, and more.

Symbol: represents a unique identifier.

Variables

Variables in JavaScript are containers that hold data values.

To declare a variable in JavaScript, you can use the var, let, or const keyword

var: declares a variable globally, or locally to a function, regardless of block scope.

let: declares a block-scoped variable, which means it only exists within the block it was declared in.

const: declares a block-scoped variable that cannot be reassigned a new value once it has been initialized.

REST API Conventions

1. Use of HTTP methods: RESTful APIs use HTTP methods (GET, POST, PUT, DELETE, PATCH) to perform operations on resources.

2. Resource-based URLs: Each resource should have a unique URL that identifies it, and the URL should only represent the resource, not the action being performed on it.

3. Use of HTTP status codes: HTTP status codes should be used to indicate the success or failure of an API call.

4. Consistent naming conventions: Resources, URLs, and other elements of the API should follow consistent naming conventions to make them easily understandable and predictable.

Clean code conventions

Clean code conventions refer to a set of best practices and guidelines for writing code that is easy to read, understand, and maintain.
These conventions focus on improving the quality of code by making it more modular, efficient, and easier to test.
Some common clean code conventions include using meaningful variable and function names,
keeping functions and classes small and focused, writing comments and documentation, and following consistent coding style.
